Einleitung

Zweck

Dieses Dokument stellt die Spezifikation des Projektes „vHike“ dar. Die Spezifikation definiert hier einen Prototypen für die spätere Benutzeroberfläche, beschreibt die Funktionsabläufe.

Motivation

Dieses Projekt ist ein Teilprojekt vom Projekt „Privacy Managment Platform“. „vHike“ sollte im Rahmen dieses Projektes als Beispiel App implementiert werden. „vHike“ sollte die Funktionalitäten von „PMP“ aufzeigen und diese in den Vordergrund stellen.

Externe Bibliotheken

Beim Einsatz werden unteranderem auch externe Bibliotheken eingesetzt. In diesem Falle ist es die GSON Bilbiothek von Google.

Funktionale Anforderungen

Use Cases

Funktionen

Registrieren

Es wird möglich sein neue Benutzer zu regestrieren. Es wird eine aktive Email-Adresse gebraucht.

Einloggen

Um sich im System einzuloggen benötigt man einen Benutzer und ein Passwort, welcher bei der Registrierung angegeben wurde.

Profil ansehen

Es wird möglich sein in eigenes Profil einzusehen und die eigene Bewertung abzufragen. Unteranderem sollte hier die Möglichkeit bestehen seine Profildaten zu ändern.

History anzeigen

Die History wird die Fahrten anzeigen, bei welchen man selbst dabei war.

Profil bewerten

Aus der History heraus wird es möglich sein andere Mitfahrer zu bewerten.

Fahrt starten

Es wird möglich sein eine Fahrt bis zu einem Ziel zu starten und die Anzahl der freien Plätzen anzugeben. Es wird eine Karte angezeigt, auf welcher man sehen kann, wo man sich gerade befindet und falls möglich Mitfahrer einladen bzw. ablehnen.

Fahrt suchen

Es wird möglich sein nach Fahrten bis zu einem Ziel zusuchen. Man wird auf eine Karte weitergeleitet und kann nun auf eine Einladung von einem Fahrer, welcher das gleiche Ziel hat, warten.

Anforderungen an vHike

Oberfläche

Die Oberfläche von vHike wird sich in die Oberfläche von Android einfügen. Dazu gehört eine nachvollziehbare und möglichst einfache Menüstruktur. Es gelten die Android User Interface Guidelines.
