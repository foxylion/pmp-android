Einleitung

Zweck

Dieses Dokument stellt die Spezifikation des Projektes InfoApp dar. Diese wird im Rahmen des Studienprojekts als eine Beispielapp für das Hauptprojekt "Privacy Management Platform" entwickelt. Das Dokument summiert die Ergebnisse der Analyse und repräsentiert die Grundlage für die spätere Entwicklung. Es ist daher eine vertragliche Grundlage für Kunde und Entwickler, um die zu entstehende Software und ihre Funktionalität so ausführlich und vollständig wie möglich zu beschreiben, damit die weitere Entwicklung stets auf dieses Dokument zurückgreifen kann.

Die Spezifikation definiert dabei einen grafischen Prototypen für die spätere Benutzeroberfläche, beschreibt die gewünschten Funktionsabläufe und definiert im Begriffslexikon die Bedeutung von speziellen Begriffen.

Projektüberblick & Motivation

Die InfoApp dient hauptsächlich als Informationsapplikation um möglichst viele Funktionalitäten von PMP darzustellen und Features zu nutzen. Dazu gehört hauptsächlich die Demonstration der Ressourcengruppen.

Leserkreis

Diese Spezifikation dient als zentrales Dokument für die folgenden Lesergruppen:

  • den Entwicklern von der InfoApp
  • den Entwicklern von PMP
  • dem Kunden
  • den Betreuern des Studienprojekts

Konventionen

In diesem Dokument werden mehrere Konventionen verwendet, um knapp, übersichtlich, lesbar und treffend zu sein:

  • Für Überbegriffe wie Benutzer oder Entwickler wird das generische Maskulinum verwendet. Da die Deutsche Sprache hier keine zufriedenstellende Alternative kennt, sind damit selbstverständlich auch alle weiblichen Leserinnen angesprochen.
  • In diesem Dokument wird die deutsche Sprache verwendet; dabei ist allerdings zu beachten, dass sich übernommene oder übersetzte Begriffe aus dem Englischen nicht vermeiden lassen. Diese werden dann jedoch grammatikalisch an die Deutsche Sprache angepasst.
  • Ein Verweis in diesem Dokument auf andere Kapitel wird so dargestellt:
  • Die Versionsnummern sind nach dem Schema aufgebaut, dass zuerst die Iterationsnummer genannt wird und anschließend die Versionsnummer der Spezifikation in dieser Iteration. "3.2" bezeichnet z.B. die 2. Version der Spezifikation für die 3. Iteration.

Funktionale Anforderungen

Anforderungen an die InfoApp

Oberfläche

Die Oberfläche der InfoApp wird sich nahtlos in die Oberfläche von Android einfügen. Dazu gehört eine konsistente und logische Anordnung der Bedienelemente genauso wie eine nachvollziehbare und möglichst einfache Menüstruktur. Es gelten die Android User Interface Guidelines, von denen nur in begründeten Einzelfällen abgewichen wird.

Standardeinstellungen

Die InfoApp wird ohne weitere Konfiguration sofort nach der Installation ("Out of the Box") einsatzbereit sein. Die einzige Bedingung ist, dass PMP installiert sein muss.

Nicht-Funktionale Anforderungen

Bedienbarkeit

Die InfoApp wird von einem durchschnittlichen Android-Nutzer intuitiv bedient werden können.

Leistungsanforderungen

Die InfoApp wird bei Verwendung eines Motorola Defy Smartphones Antwortzeiten von unter einer Sekunde aufweisen. Sollte eine Antwort länger andauern, wird der Benutzer durch eine Verarbeitungs- oder Fortschrittsanzeige in Kenntnis gesetzt.

Portabilität & Kompatibilität

Die InfoApp wird kompatibel zur aktuellen PMP Version sein (mindestens PMP v3.x), sowie zu allen Android-Systemen der Versionen 2.x (ab Version 2.1.1 aufwärts).

Internationalisierung

Die einzige verfügbare Sprache für InfoApp ist Englisch, wobei weitere Übersetzungen ohne Anpassung des Quellcodes hinzugefügt werden können.

Installation

Die InfoApp wird im apk-Format ausgeliefert und wird manuell auf Androidsystemen installiert.

Wartbarkeit

Es wird auf einen modularisierten Aufbau der einzelnen Komponenten geachtet, der Quelltext wird nach Javadoc-Konventionen kommentiert.

Abgeschlossenheit

Die InfoApp hat außer PMP und den darin verwendeten Ressourcengruppen keine weiteren externen Abhängigkeiten.

Gesetzliche Einschränkungen

Es gibt keine generellen gesetzlichen Richtlinien oder Normen, die eingehalten werden müssen. Die Universität Stuttgart wird ein zeitlich und personell unbegrenztes Nutzungsrecht an der Software erhalten. Die InfoApp wird unter der Open-Source Software Lizenz "Apache License 2.0" freigegeben.

Benutzeroberfläche

Das Start-Panel kann vom Benutzer in den Einstellungen der InfoApp festgelegt werden.

Start-Panel

Nach dem Starten der InfoApp wird automatisch das Start-Panel (hier: Wetter-Panel) geöffnet.
Grundsätzlich wird in der InfoApp eine Kopfzeile angezeigt, in der unter anderem der Name des gerade aktiven Panels angezeigt wird.

Durch eine Wisch-Geste auf dem Bildschirm kann zwischen den einzelnen Panels hin- und hergewechselt werden.

Auf jedem Panel werden bestimmte Ressourcengruppen verwendet. Die Anzahl der verfügbaren Panels richtet sich nach den installierten Ressourcengruppen und den gewährten Rechten.
Eine Übersicht über alle Panels befindet sich im Kapitel Panels.

Mit der "Menü"-Taste können die Einstellungen der InfoApp geöffnet werden.

Zu jeder Ressourcengruppe werden die einzelnen Privacy Settings erläutert und bei jedem Privacy Setting die möglichen Werte aufgelistet.

Begriffslexikon

App

BegriffApp
SynonymeApplikation
BedeutungEine App ist eine lauffähige Software, die auf einem Android-System ausgeführt wird. Eine App definiert mindestens ein Service Feature, das mehrere Services enthalten kann. Eine App kann über eine Schnittstelle Ressourcengruppen über PMP benutzen. Eine App kann eigene Ressourcengruppen in die PMP einfügen.
BeispieleDie Demo-Szenarien, die entwickelt werden, sind Apps.
AbgrenzungNicht unter den Begriff App fällt zur PMP inkompatible, nicht-ausführbare oder nicht-mobile Software.
GültigkeitEine App erlangt Bedeutung für die PMP, sobald über sie Ressourcengruppen oder Kontexte importiert werden oder sie in PMP für Ressourcenzugriffe eingebunden wird. Die App wird wieder irrelevant, sobald der Import aufgehoben wird oder die App deinstalliert wird.
BezeichnungEine App wird durch ihren Namen identifiziert und durch ihre Funktionalität beschrieben.

Privacy Setting

BegriffPrivacy Setting
SynonymePrivacy Flag, Privacy Level
BedeutungEin Privacy Setting ist eine Ressourcen-Einstellung in PMP, die durch die Ressourcengruppe definiert wird. Ein Privacy Setting kann kontext-abhängig sein. Seine konkrete Konfiguration für eine App wird beim Zugriff auf eine Ressource der Ressourcengruppe mitübergeben. Damit kann die Ressource über die Privatsphäre der zur Verfügung gestellten Inhalte entscheiden.
BeispieleDie Ressource "Schachspiel-Highscore-Datenbank" könnte Privacy Settings wie "Lesen" oder "Schreiben" anbieten.
AbgrenzungEin Privacy Setting ist weder ein Service Feature noch ein Preset.
GültigkeitDas Privacy Setting bleibt gültig, solange die entsprechende Ressourcengruppe in PMP eingebunden ist.
BezeichnungDas Privacy Setting wird durch die zugehörige Ressourcengruppe und seinen Namen eindeutig identifiziert.

Ressource

BegriffRessource
Synonyme-
BedeutungEine Ressource ist eine konkrete Funktions- oder Daten-Schnittstelle, deren Privatsphäre reguliert werden soll.
BeispieleDie Ressourcengruppe "E-Mail-Zugriff" könnte die Ressourcen "Absender", "Empfänger", "Betreff", "Inhalt" und "Anhang" enthalten.
AbgrenzungEine Ressource ist keine Ressourcengruppe. Eine Ressource ist nach Definition auch keine allgemeine, eingebundene Datei.
GültigkeitDie Ressource bleibt gültig, solange die entsprechende Ressourcengruppe in PMP eingebunden ist.
BezeichnungDie Ressource wird durch die zugehörige Ressourcengruppe und seinen Namen eindeutig identifiziert.

Ressourcengruppe

BegriffRessourcengruppe
SynonymeRessourcenkategorie
BedeutungEine Ressourcengruppe ist eine sinnverwandte Zusammenfassung einzelner Ressourcen.
BeispieleEs könnte eine Ressourcengruppe zum Zugriff auf das Dateisystem, auf GPS-Empfang oder auf einen Web-Service geben.

GültigkeitDie Ressourcengruppe bleibt gültig, solange die entsprechende App in PMP eingebunden ist.

Service Feature

BegriffService Feature
Synonyme-
BedeutungEin Service Feature ist eine Menge von Services, die bereitgestellt werden unter der Voraussetzung, dass eine Menge von Privacy Settings erfüllt wird.
BeispieleDie App "Kalender" könnte die Service Features "Kalendereinträge lesen", "Kalendereinträge schreiben" und "Kalendereinträge per E-Mail versenden" anbieten. Dabei werden jeweils höhere Einstellungen der Privacy Setting verschiedener Ressourcengruppen verlangt, aber dafür auch mehr Services angeboten.
AbgrenzungEin Service Feature ist weder ein Privacy Setting noch ein Preset.
GültigkeitEin Service Feature bleibt gültig, solange die entsprechende App in PMP eingebunden ist. Es können mehrere Service Features gleichzeitig aktiviert sein.
BezeichnungDas Service Feature wird durch die zugehörige App und ihren Namen eindeutig identifiziert.
UnklarheitenDas aus der Spezifikationsversion 1.x bekannte Service Level wird vom Service Feature abgelöst.
