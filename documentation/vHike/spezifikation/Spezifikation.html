Einleitung

Zweck

Dieses Dokument stellt die Spezifikation des Projektes „vHike“ dar. Die Spezifikation definiert hier einen Prototypen für die spätere Benutzeroberfläche und beschreibt dessen Funktionsabläufe.

Motivation

Dieses Projekt ist ein Teilprojekt vom Projekt „Privacy Managment Platform“. „vHike“ sollte im Rahmen dieses Projektes als Beispiel App implementiert werden. „vHike“ sollte die Funktionalitäten von „PMP“ aufzeigen und diese in den Vordergrund stellen.

Externe Bibliotheken

Beim Einsatz werden unteranderem auch externe Bibliotheken eingesetzt. In diesem Falle ist es die GSON Bilbiothek von Google.
- gson-2.0.jar

Funktionale Anforderungen

Use Cases

Funktionen

Registrieren

Es wird möglich sein neue Benutzer zu registrieren. Es wird eine aktive Email-Adresse benötigt. Anzugeben sind eine aktive Email-Adresse, Benutzername und Passwort. Optional sind Name, Vorname, Beschreibung (?) und Telefonnummer (?) (optional/pflicht?).

Einloggen

Um sich im System einzuloggen benötigt man einen Benutzer und ein Passwort, welcher bei der Registrierung angegeben wurde. Über anhaken des entsprechenden Radio-Buttons erfolgt die Anmeldung automatisch.

Ausloggen

Über das Hauptmenü kann aus vHike ausgeloggt werden. Nach dem Logout wird der Login-Screen angezeigt.

Profil ansehen

Es wird möglich sein in eigenes Profil einzusehen und die eigene Bewertung abzufragen. Unteranderem sollte hier die Möglichkeit bestehen seine Profildaten zu ändern.

Profile anderer Benutzer lassen sich während einer Fahrt über Klicken des Benutzernamen öffnen.

History anzeigen

Die History wird die Fahrten anzeigen, bei welchen man selbst dabei war. Entweder als Fahrer oder Mitfahrer. Eine Fahrt gibt Datum und Zielstadt an. Je Fahrt werden Fahrer und alle Mitfahrer angezeigt.

Profil bewerten

Aus der History heraus wird es möglich sein andere Benutzer zu bewerten. Die Bewertung kann pro Profil nur einmal duchgeführt werden.

Fahrt starten (WLAN)

Es wird möglich sein eine Fahrt bis zu einem Ziel zu starten und die Anzahl der freien Plätzen anzugeben. Zudem ist es möglich Zwischenziele hinzuzufügen. Dabei werden der Standort bestimmt, als auch automatisch nach potentiellen Mitfahrer innerhalb eines festgelegten Umkreises gesucht. Gefundene Mitfahrer werden auf der Karte angezeigt und über eine Statusbar-Benachrichtigung geworfen. Zusätzlich wird ein Slider geöffnet, der alle gefundenen Mitfahrer anzeigt.

Mitfahrer einladen/verwerfen (WLAN)

Sind Benutzer in dem Slider hinzugefügt worden, können diese über Klicken eines grünen Häkchens eingeladen oder eines roten Kreuzes verworfen werden. Im Falle einer Einladung verändert sich das Häkchen zu einem gelben Fragezeichen und wartet nun auf eine Interaktion des Eingeladenen. Wird ein Benutzer verworfen wird wird dieser aus der Liste als auch von der Karte entfernt und kann erst wieder bei einer neu angelegten Fahrt gefunden werden.

Fahrt suchen (WLAN)

Es wird möglich sein nach Fahrten zu einem Ziel zu suchen. Zwischenziele sind nicht möglich. Man wird auf eine Karte weitergeleitet und kann nun auf eine Einladung von einem Fahrer, welcher das gleiche Ziel hat, warten.

Einladung akzeptieren/ablehnen (WLAN)

Ein Einladung kann entweder abgelehnt oder angenommen werden. Wird ein Fahrer abgelehnt verschwindet er von der Karte und kann von diesem nicht mehr eingeladen werden und erst wieder nach neu Anlegen einer Suche gefunden werden. Wird die Einladung akzeptiert (In Auto einsteigen), wird die Karte geschlossen und man kehrt zum vorherigem Menü zurück. Zudem wird eine Statusbar-Benachrichtigung geworfen. Diese öffnet einen Dialog mit dem sich das Aussteigen aus einem Auto simuliert wird.

Weg einzeichnen

Für den Fahrer besteht die Möglichkeit eine Route von seinem momentanen Standort bis zu einem Zielort oder Fahrt-Suchenden einzeichnen zu lassen. Mehrere Routen, zum Beispiel zum Zielort und zu einem Fahrt-Suchenden wird nicht unterstützt.

Über Telefon/SMS/Email kommunizieren

vHike ermöglicht eine Kommunikation zwischen Fahrer und potentiellen Mitfahrern über einen Anruf, SMS oder Email. Dies wird ermöglicht über das Öffnen eines Benutzerprofils oder über einen entsprechenden Dialog, der sich über Klicken eines Suchenden-Icon öffnet.

Diese Funktion steht nicht zur Verfügung, falls ein Benutzer das Service Feature "Anonymous Profile" aktiviert hat.

Mitfahrgelegenheit anlegen

Bei einer Änderung des Datums können Fahrten vorausgeplant werden. Ein Datum, welches in der Vergangenheit liegt, wird nicht akzeptiert und wird durch eine Fehlermeldung abgefangen. Es sind Zielort, Datum und Anzahl der Sitplätze anzugeben. Optional können Zwischenziele angegeben werden.

Observationsmodus für Eltern

Der Observationsmodus ist eine Webbasierte-Internet-Informationsseite, welche Informationen über eigene Aktivitäten liefert. Diese Seite ermöglicht es den Mitglieder ihre eigenen Fahrten zu kontrollieren. Die Seite kann nur mit einem aktiven Benutzer betretten werden.

Statistik anzeigen

Die statistischen Auswertungen können über einen Internetbrowser eingesehen werden. Hier erscheinen Informationen über die folgenden Werte:

  • Anzahl der Fahrten
  • abgeschloßene Fahrten
  • durchschnittliche Fahrtzeit
  • Fahrten pro Monat/Jahr/Quartal
  • offene Fahrten
  • teilgenommene Fahrten als Fahrer
  • teilgenommene Fahrten als Mitfahrer
  • Mitglieder

Hauptmenü

Loginmenü der App vHike.

Hier hat der Benutzer die Möglichkeit sich einzuloggen, oder über einen neuen Benutzer anzulegen.

Registrieren

Registrieren eines neuen Benutzers.

Hier ist es möglich einen neuen Benutzer anzulegen.

Hauptmenü

Hauptmenü der App vHike.

Von hier kommt man zu verschiedenen Funktionen von der App "vHike".

Es können folgende Ansichten erreicht werden:

Fahrten

Fahrtenansicht.

Hier kann man neue Fahrten starten und nach Fahrten suchen, nachdem man sein Ziel und die verfügbaren Plätze als Fahrer bzw. die benötigte Sitzanzahl, festgelegt hat.

Profil

Profilansicht.

Hier kann man in sein eigenes Profil einsehen so wie über den "Menü-Button" sein Profil editieren

History

Historyansicht.

Hier kann man vergangene Fahrten ansehen und daraus die Benutzer bewerten.

Einstellungen

In den Einstellungen kann man die Ressourcen verwalten, dabei wird man auf die App PMP weitergeleitet.
