Einleitung

Zweck

Dieses Dokument stellt die Spezifikation des Projektes „vHike“ dar. Die Spezifikation definiert hier einen Prototypen für die spätere Benutzeroberfläche und beschreibt dessen Funktionsabläufe.

Motivation

Dieses Projekt ist ein Teilprojekt vom Projekt „Privacy Managment Platform“. „vHike“ sollte im Rahmen dieses Projektes als Beispiel App implementiert werden. „vHike“ sollte die Funktionalitäten von „PMP“ aufzeigen und diese in den Vordergrund stellen.

Externe Bibliotheken

Beim Einsatz werden unteranderem auch externe Bibliotheken eingesetzt. In diesem Falle ist es die GSON Bilbiothek von Google.
- gson-2.0.jar

Funktionale Anforderungen

Use Cases

Funktionen

Registrieren

Es wird möglich sein neue Benutzer zu registrieren. Es wird eine aktive Email-Adresse benötigt. Anzugeben sind eine aktive Email-Adresse, Benutzername und Passwort. Optional sind Name, Vorname, Beschreibung (?) und Telefonnummer (?) (optional/pflicht?).

Einloggen

Um sich im System einzuloggen benötigt man einen Benutzer und ein Passwort, welcher bei der Registrierung angegeben wurde. Über anhaken des entsprechenden Radio-Buttons erfolgt die Anmeldung automatisch.

Ausloggen

Über das Hauptmenü kann aus vHike ausgeloggt werden. Nach dem Logout wird der Login-Screen angezeigt.

Profil ansehen

Es wird möglich sein in eigenes Profil einzusehen und die eigene Bewertung abzufragen. Unteranderem sollte hier die Möglichkeit bestehen seine Profildaten zu ändern.

History anzeigen

Die History wird die Fahrten anzeigen, bei welchen man selbst dabei war. Entweder als Fahrer oder Mitfahrer. Eine Fahrt gibt Datum und Zielstadt an. Je Fahrt werden Fahrer und alle Mitfahrer angezeigt.

Profil bewerten

Aus der History heraus wird es möglich sein andere Mitfahrer zu bewerten. Die Bewertung kann pro Profil nur einmal duchgeführt werden.

Fahrt starten (WLAN)

Es wird möglich sein eine Fahrt bis zu einem Ziel zu starten und die Anzahl der freien Plätzen anzugeben. Zudem ist es möglich Zwischenziele hinzuzufügen. Dabei werden der Standort bestimmt, als auch automatisch nach potentiellen Mitfahrer innerhalb eines festgelegten Umkreises gesucht. Gefundene Mitfahrer werden auf der Karte angezeigt und können eingeladen werden. Nach einer angenommen Einladung wird der Fahrer benachrichtigt und kann ihn über einen Button-Klick abholen.

Fahrt suchen (WLAN)

Es wird möglich sein nach Fahrten bis zu einem Ziel zusuchen. Zwischenziele sind nicht möglich. Man wird auf eine Karte weitergeleitet und kann nun auf eine Einladung von einem Fahrer, welcher das gleiche Ziel hat, warten. Ein Einladung kann entweder abgelehnt oder angenommen werden. Wird ein Fahrer abgelehnt verschwindet er von der Karte und kann von diesem nicht mehr eingeladen werden. Wird die Einladung akzeptiert, wird die Karte geschlossen und man kehrt zum vorherigem Menü zurück.

Mitfahrgelegenheit anlegen

Bei einer Änderung des Datums können Fahrten vorausgeplant werden. Ein Datum, welches in der Vergangenheit liegt, wird nicht akzeptiert und wird durch eine Fehlermeldung abgefangen. Es sind Zielort, Datum und Anzahl der Sitplätze anzugeben. Optional können Zwischenziele angegeben werden.

An Mitfahrgelegenheit teilnehmen

Über das Hauptmenü gelangt man in eine Übersicht, in der alle vorausgeplanten Fahrten mit Datum, Zielort(e) und Anzahl der freien Sitzplätze angezeigt werden. Nach auswählen einer Fahrt kann nach einem freiem Platz angefragt werden. (evtl. ohne auswählen direkt in der Auswahl anfragen?)

Anforderungen an vHike

Oberfläche

Die Oberfläche von vHike wird sich in die Oberfläche von Android einfügen. Dazu gehört eine nachvollziehbare und möglichst einfache Menüstruktur. Es gelten die Android User Interface Guidelines.

Service Features

Ressource GPS

Die GPS-Ressource gibt dem Benutzer die Möglichkeit, GPS-Daten nach einem Zeitintervall zu aktualisieren oder in einem vordefinierten Umkreis zu suchen.

Genauigkeit

Wird dieses Service Feature aktiviert, werden genauere Daten wie Stadt, Straße und [...] zum Standort bestimmt.

Position-Update-Intervall

Nach einem bestimmten Zeitintervall soll die Position, in Latitude und Longitude, aktualisiert werden.

Position-Update-Distance

Die Position wird erst aktualisiert, falls immer wieder eine festgelegte Meteranzahl zurückgelegt wurde.

Route-Update-Intervall

Nach einem bestimmten Zeitintervall soll die Route automatisch immer wieder neu eingezeichnet werden.

Search-Perimeter-Intervall

vHike sucht für den Fahrer in einem vordefinierten Umkreis in Kilometer nach Mitfahrern.

Check-Offer-Intervall

vHike informiert den Mitfahrer über aktuelle Einladungen von Fahrern nach einem vordefinierten Zeitintervall.

Search-Passenger-Intervall

vHike sucht für den Fahrer in einem vordefinierten Zeitintervall nach Mitfahrern.

Ressource vHike

Die vHike-Ressource beinhaltet alle Funktionen die mit dem Webservice in Verbindung stehen. vHike benötigt diese Ressource, um sich An- und Abzumelden, zum Bewerten, Profil ansehen, Fahrt starten, Fahrt suchen, Mitfahrgelegenheit anlegen und an Mitfahrgelegenheit teilnehmen. Diese Ressource stellt zudem Service Feature bereit Art der Benachrichtigung festzulegen und sein eigenes Profil zu anonymisieren.

Search by Rating (raus?)

vHike zeichnet nur gefundene Mitfahrer/Fahrer ein, die eine bestimmte Bewertung besitzen. Die Bewertung wird in Sternen dargestellt. Ein Stern ist das Minimum, fünf Sternen das Maximum.

Anonymous-Profile

Wird dieses Service Feature aktiviert, werden alle Informationen ausser Benutzername und Bewertung, für andere ausgeblendet.

Notification

Eine Benachrichtigung kann über einen Anruf, SMS, Email (?) oder durch eine Statusbarnachricht erfolgen.
